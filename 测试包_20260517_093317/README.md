# Pieris编辑部

多角色 AI 小说编辑部体验站。上传你的小说 TXT/EPUB 文件，选择一位编辑角色获得长篇锐评，同时左侧群聊窗口会展开 2-6 位编辑的内部讨论，并可额外生成一份结构化的作品体检报告。

## 比赛版亮点

- **主编长评**：用不同角色视角给出鲜明、可区分的长评
- **编辑群聊**：让多位编辑在同一稿件上展开讨论
- **作品体检报告**：从结构、人物、节奏、语言、可读性五个维度快速看清稿件状态
- **扫码即用**：支持服务端演示模式，评委无需填写自己的 API Key
- **PWA**：可添加到桌面，现场演示更像一个独立产品

## 长评策略

- 所有角色都必须同时关注局部证据与整体判断，不再只盯局部引文。
- 长评会稳定覆盖整体框架、题材处理、文风气质、完成度等锚点。
- 三位专业编辑更强调编辑视角、问题优先级与修改方案。
- 其余角色与新增角色都保持“读者型编辑”定位，强调阅读体验、人物偏好与题材感受。
- 输出长度不做硬限制，以把判断说透为准。

## 群聊讨论

左侧「内部讨论群」窗口模拟 QQ 群聊，支持 2-6 位编辑参与（默认铁板、余墨、知苑）。群聊按 3 轮批量推进，每轮一次生成多人消息，并把前几轮记录注入下一轮，尽量保持有来有回；当主评先结束时，群聊会同步收束，并用 `……` 补全未轮到的席位。

## 本地展示资源

- `所需图片/` 已接入本地展示所需角色图片：`丰川祥子.jpg`、`克莱恩.jpg`、`贴吧哥.jpg`

## 作品体检报告

- 点击 `生成体检报告` 可独立获得一份结构化结果页
- 报告会输出：
  - 题材判断
  - 文风气质
  - 核心抓手
  - 五维评分：结构 / 人物 / 节奏 / 语言 / 可读性
  - 当前亮点 / 主要风险 / 优先修改项
  - 推荐继续查看的编辑角色

## 启动步骤

### 1. 安装依赖

```bash
pip install -r requirements.txt
```

### 2. 启动

```bash
python app.py
```

服务默认监听 `5002` 端口。

### 3. 打开浏览器访问

```
http://localhost:5002
```

## 使用流程

1. 如未启用演示模式，先输入 DeepSeek API Key，点击 **「验证」**
2. 上传 TXT 或 EPUB 文件（TXT 自动检测编码，EPUB 自动解析章节）
3. 从章节列表中选择要点评的章节
4. 点击一位编辑角色卡片作为主评；可右键或 `Ctrl`/`Cmd` 点击追加群聊成员
5. 点击 **「开始锐评」**，主评流式输出，左侧群聊同步开始
6. 可点击 **「生成体检报告」** 获得结构化诊断结果
7. 可点击 **「换人重评」** 用其他编辑重新评价（群聊也会重新开始）

## 接口说明

| 端点 | 方法 | 说明 |
|------|------|------|
| `/` | GET | 前端页面 |
| `/upload` | POST | 上传 TXT/EPUB 文件，返回章节列表 |
| `/chat/stream` | POST | SSE 流式主评 |
| `/chat/group` | POST | SSE 流式群聊 |
| `/report/health` | POST | 生成结构化作品体检报告 |
| `/demo-status` | GET | 返回当前是否启用演示模式及剩余额度状态 |
| `/verify-key` | POST | 验证 API Key |
| `/characters` | GET | 列出所有可用角色 |

## 技术栈

- 后端：Flask + Flask-CORS
- AI：DeepSeek API（deepseek-v4-pro）
- 编码检测：chardet
- EPUB 解析：ebooklib
- 前端：纯静态 HTML/CSS/JS（本地 vendored 资源，无远程 CDN 依赖），SSE 流式传输
- 部署：Gunicorn + Render

## 演示模式

为了比赛现场的二维码直达体验，项目支持服务端演示模式。

### 环境变量

参考 `.env.example`：

```bash
DEMO_MODE_ENABLED=true
DEMO_API_KEY=sk-your-deepseek-demo-key
DEMO_MAX_REQUESTS_PER_IP=8
DEMO_MAX_REQUESTS_TOTAL=80
```

### 说明

- 用户自己填写 `API Key` 时，优先使用用户提供的 Key
- 用户未填写 Key 且演示模式开启时，后端自动使用 `DEMO_API_KEY`
- 项目方提供的演示 Key 仅用于比赛展示，额度有限，先到先得
- 该演示额度不保证长期持续开放，不建议把它当成长期公共服务
- 当演示额度耗尽时，前端会提示用户改填自己的 Key 或联系项目作者现场演示
- 页面会保留可折叠的 `API 设置` 入口，用户可随时切换为自己的 Key

## Render 部署

### 1. 推送代码到 GitHub

- 新建一个 GitHub 仓库
- 把当前项目推上去

### 2. 在 Render 新建 Web Service

- 连接你的 GitHub 仓库
- Render 会自动识别 `render.yaml`
- 或手动填写：
  - Build Command: `pip install -r requirements.txt`
  - Start Command: `gunicorn -b 0.0.0.0:$PORT app:app`

### 3. 在 Render 配置环境变量

- `DEMO_MODE_ENABLED=true`
- `DEMO_API_KEY=你的 DeepSeek Key`
- `DEMO_MAX_REQUESTS_PER_IP=8`
- `DEMO_MAX_REQUESTS_TOTAL=80`

### 4. 获取线上 URL

- 部署完成后，Render 会给出一个 HTTPS 链接
- 用这个链接生成二维码，贴到海报上即可

## 海报二维码建议

- 二维码直接指向线上首页，不要指向仓库
- 海报可补一句小字：`比赛演示额度有限，先到先得；长期使用请填写自己的 API Key`
- 海报卖点建议写成：
  - 多角色 AI 小说编辑部
  - 主评 + 群聊 + 作品体检报告
  - 扫码即用
